# SynthID Provenance ŌĆö AI-Generated Image Disclosure

How Vince tracks and surfaces AI provenance for every generated image in the media library.

---

## What This Is

Every image Vince generates is written to the `media` table with SynthID provenance fields set at insert time. These fields power the **Synthid Watermark Detection** panel in the media preview UI, showing viewers that an image was AI-generated and which model produced it.

This is not watermark *detection* ŌĆö it is watermark *declaration*. Because Vince is the one generating the images, provenance is known with certainty. Setting `synthid_confidence: 1.0` is more honest than asking a model to guess whether its own output contains an invisible watermark.

---

## Fields Written Per Generated Image

| Column | Value | Notes |
|---|---|---|
| `synthid_detected` | `true` | Always true for AI-generated images |
| `synthid_confidence` | `1.0` | 100% ŌĆö we generated it |
| `synthid_generated_by` | Model ID string | e.g. `gemini-3.1-flash-image-preview` |
| `synthid_details` | `'AI-generated by Vince Creative Studio'` | Human-readable label |

---

## Generation Paths That Set These Fields

| Edge Function | Tool / Path | Model |
|---|---|---|
| `brand-prompt-agent` | `generateHeadshotScene()` | `gemini-3.1-flash-image-preview` |
| `generate-creative-image` | All generation types | `modelUsed` (dynamic, from DB config) |
| `generate-creative-package` | `registerMediaImage` call | `gemini-3.1-flash-image-preview` |
| `generate-brand-card-images` | `registerMediaImage` call | `gemini-3-pro-image-preview` |
| `generate-studio-welcome-images` | `registerMediaImage` call | `gemini-3-pro-image-preview` |

The shared helper at `supabase/functions/_shared/media-registration.ts` handles the insert for `generate-creative-package`, `generate-brand-card-images`, and `generate-studio-welcome-images`. Pass `modelUsed` to `registerMediaImage` and the SynthID fields are set automatically.

---

## UI Display

**File:** `src/components/media/MediaPreview.tsx`

The info pane (opens by default) shows a **Synthid Watermark Detection** section when `synthid_detected !== null`:

- Green badge: **Watermark Detected** with a check icon
- Confidence progress bar
- Generated by: model name
- Details text
- Collapsible "What is SynthID?" explanation

---

## What Is NOT Covered

- **Manually uploaded images** ŌĆö no SynthID data is set on upload. The field stays `null`, and the panel does not render.
- **Video generations** ŌĆö `generate-creative-video` does not insert to the `media` table directly; videos appear in `creative_studio_generations` only.
- **Backfill of historical images** ŌĆö images generated before this change (2026-03-14) have `null` values. The UI handles this gracefully (panel hidden when `synthid_detected === null`).
