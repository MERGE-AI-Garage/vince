## What Changed

The original competitive intelligence card showed a summary, weaknesses list, and a plain counter brief. This update upgrades the entire flow with five additions: scene-level video breakdown, 3 selectable campaign directions before generation, clickable "Build These" counter deliverable buttons, a new self-critique mode where Vince watches himself, and full rendering of all self-critique fields including narrative issues.

## Feature 1: Scene-Level Breakdown

### What it does
`analyze-competitor-video` now requests a scene-by-scene breakdown from Gemini 2.0 Flash. Each scene includes a timestamp range, scene type label, emotional signal, and the marketing intent behind it.

### Schema
```typescript
scenes: Array<{
  timestamp: string;        // "0:00–0:06"
  scene_type: string;       // "Product reveal", "Performance demo", etc.
  emotional_signal: string; // "anticipation", "awe", "urgency"
  marketing_intent: string; // "Establish power/performance positioning"
}>
```

### UI
Compact timeline inside the orange card — monospace timestamp, scene type, emotion right-aligned. Rendered only if Gemini returns scenes. No images, just text.

## Feature 2: "Beat This Ad" + 3 Campaign Directions

### What it does
Replaces the plain "Competitive Intel" header with a "Beat This Ad" framing (Target icon). Shows 3 distinct campaign direction options generated by Gemini — each with a title, concept description, emotional angle, and punchy tagline.

The counter brief is still present, collapsed under `<details>`.

### Schema
```typescript
campaign_directions: Array<{
  title: string;           // "AI That Actually Helps" — 3-5 words
  concept: string;         // 1-2 sentence campaign description
  emotional_angle: string; // "Relatability over aspiration"
  tagline: string;         // "Smart is the new fast."
}>
```

### Selection flow
1. Vince analyzes competitor video → orange card with 3 direction cards
2. User clicks a direction
3. `handleDirectionSelect()` sends: `"Let's go with the '[Title]' direction. [concept] Tagline: '[tagline]'"`
4. Vince interprets as intent → calls `generate_creative_package` with the direction as brief context

Direction selection goes through Vince's message flow rather than a direct edge function call. This keeps Vince in the loop to add creative nuance and narrate the generation.

## Feature 3: Counter Deliverables — "Build These" Buttons

### What it does
`counter_deliverables` was already being returned by `analyze-competitor-video` and extracted into the frontend type, but was never rendered anywhere. It now appears as a "Build These" section in the orange card — one clickable button per suggested deliverable.

### What the data looks like
```typescript
counter_deliverables: Array<{
  name: string;             // "Hero Banner", "Social Story", "LinkedIn Post"
  description: string;      // What this specific asset should show/say
  deliverable_type: string; // "display_banner", "social_story", "linkedin_post"
  aspect_ratio: string;     // "16:9", "9:16", "4:3"
}>
```

### UI
Each deliverable renders as a button: asset name on the left, aspect ratio label on the right. Clicking sends:
```
"Build a [Hero Banner] for the counter-campaign: [description]"
```
Vince receives that, confirms with the user, and calls `generate_creative_package` with the deliverable type and description as context.

### Why it goes through Vince
Direct edge calls would bypass Vince's confirmation step and the conversational context. Keeping it as a message lets Vince acknowledge the selection, potentially adjust the brief, and narrate while generation runs — which matters especially in voice mode.

## Feature 4: Vince Watching Himself (`analyze_self_demo`)

### What it does
New Vince tool. When you share a YouTube link to a Brand Lens / Vince demo recording, Vince watches it as a product designer, UX critic, and AI engineer combined — analyzing demo effectiveness, not competitor marketing strategy.

The same `analyze-competitor-video` edge function handles this via `mode: 'self_critique'`. No new edge function needed. `brand_id` is not required in self-critique mode.

### Tool definition (in `brand-prompt-agent`)
```typescript
{
  name: 'analyze_self_demo',
  description: 'Watch a screen recording of a Brand Lens / Vince demo and analyze it as a product designer and UX critic. Use when the user shares a YouTube link to a Brand Lens demo recording and wants feedback on the product experience, demo flow, or workflow improvements.',
  parameters: {
    properties: {
      video_url: { type: 'string', description: 'YouTube URL of the Brand Lens demo recording' }
    },
    required: ['video_url']
  }
}
```

### Edge function: self_critique mode
When `mode === 'self_critique'`, the edge function:
- Skips brand context injection entirely
- Uses a product-design–focused system prompt:
  ```
  You are watching a screen recording demo of an AI creative director product called Vince (Brand Lens).
  Analyze it as a product designer, UX critic, creative director, and AI engineer combined.
  Be direct, specific, and actionable. Cite timestamps where possible.
  ```
- Returns a `SelfCritiqueAnalysis` object, not a `CompetitiveAnalysis`

### What Gemini returns
```typescript
{
  product_summary: string;            // 2-3 sentence overall impression
  demo_score: number;                 // 0-100 overall demo effectiveness
  ux_observations: string[];          // Friction, confusion moments (with timestamps)
  missed_opportunities: string[];     // Features not shown, better flows that were skipped
  demo_narrative_issues: string[];    // Pacing problems, unclear explanations, weak hooks
  recommended_improvements: string[]; // Specific actionable fixes
}
```

### How to trigger it
Tell Vince: `"Watch this demo recording and tell me what you think: [YouTube URL]"`

Works in text chat and voice mode. In voice mode, paste the URL into the URL input field in the voice bar.

### Practical use
1. Record a run-through (QuickTime, Loom, OBS — anything)
2. Upload to YouTube (unlisted is fine)
3. Give Vince the URL
4. Use the score and structured critique to improve demo pacing, fix UX friction, or update the system prompt
5. Repeat before each major rehearsal leading up to the submission deadline

This is a development tool — not intended for the submission demo itself. It's a strong DevPost narrative ("the AI creative director critiques its own demo footage to improve itself") but showing it in the 4-minute demo video would cost time better spent on the core voice-to-creative-package loop.

## How to Test

### Test 1: Beat This Ad — scene breakdown + directions + deliverable buttons
1. Have a brand active (e.g. either seeded demo brand)
2. In Vince chat: `Analyze this competitor ad: [YouTube URL of any real ad]`
3. Wait ~15–30s for Gemini to process the video
4. **Expect the orange card to show all of:**
   - Target icon + "Beat This Ad" header
   - Scene timeline section with timestamps, scene type, emotion label
   - "Strategic Openings" bullet list
   - "3 Ways to Beat It" — 3 direction cards with title, concept, tagline
   - "Build These" — buttons for each counter deliverable (Hero Banner, Social Story, LinkedIn Post, etc.) with aspect ratio label
   - "Full Counter Brief" collapsed at the bottom
5. **Test direction selection:** Click one direction card → message sends in format `"Let's go with the '[Title]' direction..."` → Vince responds and calls `generate_creative_package`
6. **Test deliverable button:** Click a "Build These" button → message sends `"Build a [name] for the counter-campaign: [description]"` → Vince confirms and generates

### Test 2: Vince watching himself — full violet card
1. Upload any screen recording of the Brand Lens workflow to YouTube (unlisted is fine; even 2-3 minutes works)
2. In Vince chat: `Watch this demo recording and tell me what you think: [YouTube URL]`
3. Wait ~15–30s
4. **Expect the violet card to show all five sections:**
   - Header with "Self Analysis" + "Demo Score: XX/100"
   - Product summary
   - UX Observations with timestamps
   - Missed Opportunities
   - Narrative Issues (pacing, hook strength, clarity problems)
   - Recommended Improvements
5. **Test in voice mode:** Say "Watch this demo" → paste URL into voice bar URL field → same violet card should appear in the chat thread

### Test 3: Voice path for competitor analysis
Same as Test 1 but in voice mode. Paste the URL into the voice bar URL field instead of the chat input. The `onToolResult` handler uses the same extraction logic, so the orange card renders identically.

### Test 4: Brand alignment score (verification)
1. Ask Vince to generate a creative package (text or voice)
2. **Expect:** After interleaved images render, alignment score appears at the bottom of `CreativePackageDisplay`
3. Score color: emerald (≥80), amber (60–79), red (<60)
4. Low score on a brand with no synthesized profile is correct behavior — run brand playbook first to raise it

### Edge cases
- If Gemini doesn't return `scenes` or `campaign_directions` (API drift, flakiness), those sections don't render — guarded with `?.length > 0`
- If a YouTube video is private or unavailable, the edge function returns an error and Vince reports it
- `analyze_self_demo` vs `analyze_competitor_content` disambiguation: if you paste a generic YouTube URL without saying "demo" or "Brand Lens", Vince may default to competitor mode. Be explicit: "Watch this Brand Lens demo recording"
- `demo_score` is 0 if Gemini returns it as null — worth watching to see if Gemini skips it on some runs

## Brand Alignment Score

Already fully wired — confirmed, not a new feature. The score is calculated in `generate-creative-package` based on brand profile completeness (visual DNA, tone of voice, directives, generation prompt). Returned as `brand_alignment: { score, dimensions }`, extracted into `AgentResponse.creative_package.brand_alignment`, passed as the `brandAlignment` prop to `CreativePackageDisplay`. Renders with emerald/amber/red color coding and dimension breakdown.
